# 🎬 Movie Recommendation Flow — Langflow

> **CineBot** is an AI-powered movie recommendation engine built with Langflow.
> Feed it your favourite genres and your current mood and it returns a rich,
> ranked list of films with ratings, director info, mood-match emojis, and
> streaming hints — all generated by an OpenAI GPT model.

---

## 📁 Files in This Package

| File | Purpose |
|------|---------|
| `flow.json` | Complete Langflow flow — import this into Langflow UI |
| `MovieRecommenderComponent.py` | Custom Python component (CineBot engine) |
| `README.md` | This guide |

---

## 🗺️ Flow Architecture

```
┌─────────────────────┐
│  Favorite Genres    │  ← TextInput node
│  (e.g. "Action,    │
│   Sci-Fi")          │
└────────┬────────────┘
         │  genres
         ▼
┌─────────────────────────────────────────┐       ┌──────────────────────────┐
│        🎬 Movie Recommender             │──────▶│  🍿 Recommendations Out  │
│          (CineBot Engine)               │       │  (Formatted Markdown)    │
│                                         │       └──────────────────────────┘
│  Inputs:  genres, mood, # of films,     │
│           era, extras, OpenAI key       │       ┌──────────────────────────┐
│  Outputs: recommendations, reasoning   │──────▶│  🧠 Reasoning Output     │
└─────────────────────────────────────────┘       │  (Per-movie rationale)   │
         ▲                                        └──────────────────────────┘
         │  mood
┌────────┴────────────┐
│  Current Mood       │  ← TextInput node
│  (e.g. "Happy")     │
└─────────────────────┘
```

### Nodes

| Node | Type | Role |
|------|------|------|
| **Favorite Genres** | TextInput | Accepts comma-separated genres from the user |
| **Current Mood** | TextInput | Accepts the viewer's current emotional state |
| **Movie Recommender** | Custom Component | Builds the LLM prompt, calls OpenAI, parses & formats results |
| **Recommendations Output** | ChatOutput | Renders the ranked movie list in markdown |
| **Reasoning Output** | ChatOutput | Renders CineBot's per-movie rationale |

## 🚀 How to Import & Run

### 1. Prerequisites

```bash
pip install langflow openai
```

### 2. Import the Flow into Langflow

1. Launch Langflow:
   ```bash
   langflow run
   # or
   python -m langflow run
   ```
2. Open your browser at **http://localhost:7860**
3. Click **"New Flow" → "Import"** (or drag-and-drop)
4. Select `flow.json` from this package
5. The flow canvas will load with all 5 nodes pre-wired

### 3. Register the Custom Component

Langflow needs to know about `MovieRecommenderComponent.py`:

**Option A – Custom components directory (recommended)**
```bash
langflow run --components-path ./
# Place MovieRecommenderComponent.py in the same folder
```

**Option B – Upload via UI**
1. In the Langflow sidebar, click **Custom Components**
2. Click **"+ Upload Component"**
3. Select `MovieRecommenderComponent.py`

### 4. Configure the Component

Click the **🎬 Movie Recommender** node and fill in:

| Field | Value | Notes |
|-------|-------|-------|
| **OpenAI API Key** | `sk-...` | Required — stored securely as a secret |
| **OpenAI Model** | `gpt-4o` | Default; swap to `gpt-3.5-turbo` for lower cost |
| **Number of Recommendations** | `5` | Between 1 and 10 |
| **Era Preference** | `Any Era` | Optional filter |
| **Additional Preferences** | *(blank)* | Optional free-text extras |

### 5. Run the Flow

1. Set **Favorite Genres** → e.g. `"Horror, Thriller"`
2. Set **Current Mood** → e.g. `"Scared / Thrilled"`
3. Click ▶ **Run**
4. The two output panels display:
   - **Recommendations Output** — a formatted movie list
   - **Reasoning Output** — CineBot's per-movie rationale

## 🧩 Component API Reference

### Inputs

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `genres` | `MessageTextInput` | `"Action, Sci-Fi"` | Comma-separated favourite genres |
| `mood` | `DropdownInput` | `"Happy"` | Viewer's current emotional mood |
| `num_recommendations` | `IntInput` | `5` | Number of films to return (1–10) |
| `era_preference` | `DropdownInput` | `"Any Era"` | Filter by cinema era |
| `additional_preferences` | `StrInput` | `""` | Free-text extras (optional) |
| `openai_api_key` | `SecretStrInput` | — | OpenAI API key (required) |
| `model_name` | `DropdownInput` | `"gpt-4o"` | GPT model to use |

### Outputs

| Name | Method | Returns |
|------|--------|---------|
| `recommendations` | `recommend_movies()` | Formatted markdown movie list |
| `reasoning` | `get_reasoning()` | Per-movie rationale & summary |

## ⚙️ Customisation Tips

### Change the number of results
Edit the **Number of Recommendations** field on the component node (1–10).

### Add a new mood
In `MovieRecommenderComponent.py`, find the `mood` `DropdownInput` and add
your value to the `options` list.

### Swap the model
Change `model_name` to `gpt-4-turbo` (better reasoning) or `gpt-3.5-turbo`
(faster & cheaper).

### Use as a Python library (without Langflow UI)

```python
from MovieRecommenderComponent import MovieRecommenderComponent

bot = MovieRecommenderComponent()
bot.genres = "Action, Sci-Fi"
bot.mood = "Excited"
bot.num_recommendations = 3
bot.era_preference = "Recent (2020+)"
bot.additional_preferences = "No sequels"
bot.openai_api_key = "sk-..."
bot.model_name = "gpt-4o"

result = bot.recommend_movies()
print(result.text)
```

## 🔒 Security Notes

- The OpenAI API key is handled via `SecretStrInput` — it is **never logged**
  or exposed in the flow JSON.
- For production deployments, use environment variables:
  ```bash
  export OPENAI_API_KEY=sk-...
  ```
  Then read it in the component via `os.environ["OPENAI_API_KEY"]`.

## 🐛 Troubleshooting

| Symptom | Fix |
|---------|-----|
| `⚠️ Could not parse LLM response as JSON` | The model returned malformed JSON. Try `gpt-4o` which supports `response_format=json_object` natively. |
| `AuthenticationError` | Check your OpenAI API key is valid and has credits. |
| Component not found in Langflow | Ensure `MovieRecommenderComponent.py` is in your `--components-path` directory. |
| Flow nodes appear disconnected | Re-import `flow.json` — do not copy-paste individual nodes. |
